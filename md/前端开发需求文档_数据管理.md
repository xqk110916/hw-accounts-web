# 模块开发需求：数据管理

## 1. 全局设计与架构规范
* **UI/UX 基准**：所有界面布局、设计规范及组件形态需严格参考现有 `src/views/task` 模块，确保系统的 UI 和交互体验保持完全一致。必须将“复用 task 模块交互规范”作为最高准则。
* **基础组件与预估依赖**：基于 Vue3 组合式 API 和 Element-Plus 标准组件库进行拆解，需保证所有页面表格分页和搜索习惯的高度收敛。
* **推测可直接复用的组件形态**：
  * **搜索表单组件**：可统一封装顶部带有各种类型筛选条件（Input、Select、DateRange、Checkbox、RadioGroup）和“查询”、“重置”操作的条件搜索组件 (`SearchFilter` 等)。
  * **表格与分页组件**：所有页面均包含通用的 Table 结构支持列展示及底部标准的分页条（共计条数、每页条数切换和页码跳转），强烈建议复用或封装为高阶表格组件。
  * **抽屉/弹窗**：所有的详情、新增、编辑操作均优先复用 `task` 模块已有的抽屉 (Drawer) 弹出交互形态，删除操作依赖统一风格的二次确认弹窗 (MessageBox)。

## 2. 界面需求细化（按图片逐一展开）

### 界面：初始录入
![界面原图](md/img/初始录入.png)

* **界面概述**：用于系统中各类数据信息（如材料信息）的导入记录查看、状态追踪及人工审批流转情况追踪。
* **视图结构拆解**：
  * 顶部：多条件搜索筛选区。
  * 列表右上侧：独立按钮功能组。
  * 主体：含有左侧复选框、右侧操作列的明细表格配置以及底部分页区。
* **数据元素与校验清单**：
    * **查询条件**：
        * 添加时间：日期范围选择框，默认空值。
        * 状态：多选框组（选项：全部、待提交、待审核、审核通过、审核拒绝），默认勾选“全部”。
    * **数据展示**：
        * 列表呈现：数据类型、导入时间、导入人、审批时间、审批人、据条数、状态、操作。
        * 校验/排版：状态列（待确认是否需配合如 tag 组件进行彩色预警标识区分，例如拒绝为红色，待审核为黄色等）。
* **操作与交互逻辑**：
    * **功能按钮**：`查询`、`重置`、`添加`（点击添加触发新增信息表单抽屉或新路由全屏表单）。
    * **行级操作反馈**（根据当前所处的“状态”动态呈现不同操作项）：
        * 【待提交】：展示 `详情`、`编辑`、`提交`、`删除`。
        * 【待审核】：展示 `详情`、`审核`。
        * 【审核通过】：展示 `详情`、`编辑`、`删除`。
        * 【审核拒绝】：展示 `详情`、`编辑`、`提交`、`删除`。
    * **删除交互**：点击均需唤起二次确认对话框 (MessageBox)。
    * **详情/编辑交互**：点击后应右侧划出详情/编辑抽屉面板（继承 Task 模块规范）。

### 界面：变化量报表
![界面原图](md/img/变化量报表.png)

* **界面概述**：聚焦出库与入库操作的变化量记录及材料流转报表。
* **视图结构拆解**：
  * 顶部折行展示多维度报表特征搜索表单。
  * 表格头部区（具 Tab 切换特性的 汇总/明细 按钮）。
  * 含有复杂排版的列表及分页器。
* **数据元素与校验清单**：
    * **查询条件**：
        * 库房：下拉单选联调（可选库房或平衡区）。
        * 材料类型：下拉单选。
        * 生产单位：文本输入。
        * 出入库时间：日期范围选择框。
        * 类型：单选框组（全部、入库、出库），默认选中全部。
    * **数据展示**：
        * 报表表头：任务编号、容器号、材料代码、生产单位、出入库时间、位置、封记编码、重量（毛/皮/净）。
        * 排版特点：无底部“操作”列及复选框列；任务编号附带 `【入库】` 或 `【出库】` 状态字符开头前缀识别；重量列用“斜杠（/）”组合形式拼接为单列。
* **操作与交互逻辑**：
    * **视图类型互动**：点击左上方文字区域 `汇总 / 明细`，会在两套不同的查询视图表格/结构间进行响应式切换。本原图正激活展示的是“明细”。
    * **数据导出功能**：点击右侧独立全局色块按钮 `导出` 执行该表单维度数据的接口异步获取及 Excel 等报表文件下载。

### 界面：存量报表
![界面原图](md/img/存量报表.png)

* **界面概述**：呈现实时材料当前静态资产的盘点与存量统计报表。
* **视图结构拆解**：
  * 顶部平铺搜索范围筛选区。
  * 特征统计看板（居中含有汇总统计及数值高亮效果的行）。
  * 全量存量清单带行级操作的底座表格及分页。
* **数据元素与校验清单**：
    * **查询条件**：库房（下拉选择）、材料类型（下拉选择）、生产单位（文本框）、入库时间（日期范围）。
    * **数据展示**：
        * 头部统计板组件：高亮包含“汇总/明细”文字（“汇总”呈焦点蓝色激活）。展示两项统计结果：`总容器数: xxx`（红色字体高亮数值）、`总重: xxx.xxxxxKG`（红色主视觉数值）。
        * 列表呈现：头部多选框列、库房、材料代码、生产单位、入库时间、入库批号、数量、重量（毛/皮/净）。
* **操作与交互逻辑**：
    * **统计切换**：点击“汇总”/“明细”，重载相关表格视图架构（图片激活为“汇总”视图）。
    * **全局导出按钮**：提供报表`导出`功能逻辑。
    * **行操作区**：`查看详情`，推演交互为点击后通过抽屉打开选定批号及容器项对应库存底层溯源详细页。

### 界面：封记台账
![界面原图](md/img/封记台账.png)

* **界面概述**：监测与管理各个容器配套“封记”实体的状态和台账录入功能。
* **视图结构拆解**：
  * 两层表单过滤区域。
  * 提供批量操作/新增入口功能的工具栏。
  * 具体明细维护的台账表格以及分页控制。
* **数据元素与校验清单**：
    * **查询条件**：封记编码（输入）、封记类型（下拉）、封记状态（单选：全部/完好/破损，默认选中全部）、登记时间（日期范围）、容器（下拉框）。
    * **数据展示**：
        * 列表字段：复选框、容器（内置业务拼接如"编号+名称+位置"）、封记1（拼接"编码【类型】"）、封记2（拼接"编码【类型】"）、登记时间、状态、操作（包含“详情、编辑、删除”三个超链接）。
* **操作与交互逻辑**：
    * **全局维护操作**：
        * `新增`：点击触发创建“封记台账记录”对话框。
        * `导入`：提供模板下载与上传文件接口的专用弹层界面。
        * `删除`（置灰预备）：依托勾选表格左侧复选框触发的批量防误操作多项删除。
    * **表行操作**：支持每条记录专属点击的`详情`展示及`编辑`回填修改功能（均采用通用抽屉范式），单行触发的`删除`交互应有强二次告警确认。

### 界面：材料管理
![界面原图](md/img/材料管理.png)

* **界面概述**：用作系统中各种核心物料和“材料”业务数据源的主数据登记/字典管理。
* **视图结构拆解**：
  * 占据一行的简单查询筛选器。
  * `新增`主体按钮。
  * 多选支持的主列数据表格区及分页。
* **数据元素与校验清单**：
    * **查询条件**：三个单纯的文本匹配输入（材料名称，材料代码，通用名称）。
    * **数据展示**：材料名称、材料代码、通用名称、常用单位、元素含量、同位素含量。
    * 注意字段级合并展示规范：如元素含量可能涉及多数据组合（A：99% B：1%），同位素含量同理。
* **操作与交互逻辑**：
    * **数据添加**：点击 `新增`，调启具备复合字段必填强校验特征（如关联计算配比相加总和等）的维护组件表单。
    * **单条行记录管理**：提供基于行级生命周期的修改及查阅支持体验（即包含三个常驻操作态按钮 `详情`、`编辑`、`删除`），需要强调删除是否具备被上下游报表和数据依赖占用的逻辑锁限制校验。

### 界面：标签模板管理
![界面原图](md/img/标签模板管理.png)

* **界面概述**：维护标签打印的板式、数量明细列表。
* **视图结构拆解**：
  * 仅拥有一组筛选项的单行极简查询界面。
  * 右上方高密集度多类型功能操作工具条。
  * 单一的列表界面及分页。
* **数据元素与校验清单**：
    * **查询条件**：仅有一个“打印时间”（日期范围组件）。
    * **数据展示**：复选框、打印时间、标签数量、备注。
* **操作与交互逻辑**：
    * **高度特征性顶部按钮功能**：
        * `导入` 和 `添加`：用于外部输入源录入，点击触发对应的导入规范弹层和录入抽屉。
        * `编辑模板`（白色按钮底）：属于系统可视化配置或高交互类跳转动作设计，用来定义后续打印时关联的真实物理模板绘制。
    * **基础行级操作**：具备 `详情`、`编辑` 与 `删除` 等全生命周期记录管理。

### 界面：综合查询
![界面原图](md/img/综合查询.png)

* **界面概述**：作为全局跨库、材料流向溯源、出入轨迹一体化分析的系统级中心检索看板。
* **视图结构拆解**：
  * 超大型且折为三行的“矩阵条件过滤器”群组区，带有双重日期筛查等复杂要求。
  * 具备高级交互拓展机制的表格增强栏组。
  * “信息稠密型”表格内容载体区。
* **数据元素与校验清单**：
    * **查询条件**（字段极多）：容器号（输入框）、生产单位（输入框）、收方单位（输入框）、材料代码（下拉选择组件）、入库时间（日期范围）、出库时间（日期范围）、位置（具有选择预置内容的文本交互组件）、调拨依据（下拉选择组件）、封记号（输入文本）以及类型过滤单选框组（囊括全部/已出库/库存分类）。
    * **数据展示**（包含特定双语态格式要求）：包含：位置、容器、入库信息（格式为换行：第一行展示"[入库编号]"，第二行展示真实发生"入库时间"）、出库信息（格式为换行："[出库编号]"和"出库时间"拼接）、材料名称、封记信息、调拨依据。
* **操作与交互逻辑**：
    * **表头管控**：点击 `列头管理`，调出表格的偏好设置控制器，允许用户勾选自定义展示出入库涉及的各个列项隐藏配置反馈（前端级设置项或保存偏好接口）。
    * **文件产生交互**：`导出`功能。
    * **只读化追踪限制**：该页面通常不涉及数据编辑等更改源数据的侵入式行为，仅有单一的最右侧 `详情` 的钻取和查看能力支持。

### 界面：账目管理
![界面原图](md/img/账目管理.png)

* **界面概述**：展示以“年”、“类型维”核心汇总的多角度数据统计主控视图呈现大盘。
* **视图结构拆解**：
  * 位于顶部的视图控制总分类导航（Tabs组件形态的菜单锚点）。
  * 具有业务相关性的定制类目搜索筛条（带有Tag支持）。
  * 特带加黑处理及统率全表的表头中央总标题（呈现大字号排版美感）。
  * 标准的只读呈现统记用数据二维表格。
* **数据元素与校验清单**：
    * **导航切换 Tab组**：内置五个并排锚向子视图：总账（当前图例所停留）、总账明细、库房账明细、生产单位明细账、材料账明细。
    * **查询条件**：
        * 年份：固定为时间维度（年份选择器或者有默认值初始化的搜索框："默认本年"，如 20XX）。
        * 材料：要求是 **支持多项勾选并回显 Tag 碎片** 的复合型下拉多选组件（例如回显“材料1 X”，“材料2 X”）。
        * 库房：单选性质的下拉框。
    * **数据展示**：
        * 大报表图示标题区：`<h2 class="text-center font-bold">20XX年总账</h2>` 随检索动作按年份字面特征动态响应变化。
        * 列表区域：本例以明细大盘为主，不包含复选框与行操作列。呈现核心汇总维：材料代码、生产单位、库房、件数、重量。
* **操作与交互逻辑**：
    * **页面响应拦截控制**：在点击上方的不同 Tabs 时，下方查询筛选项内容和呈现表格表头列组合需做“热切换重载”，但可不必离开当前总路由。
    * **辅助输出工具**：右上角提供标准支持 `导出` 图表汇总至本地台账数据的文件流动作。
