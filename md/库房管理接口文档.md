## 一、位置地图管理

### 1.1 层级节点管理

| 序号 | 接口名称 | 接口地址 | 请求方式 |
|:---:|---------|---------|:-------:|
| 1 | 查询所有节点（树形结构） | `/locationMap/hierarchy/tree` | GET |
| 2 | 根据平衡区ID查询树形结构 | `/locationMap/hierarchy/treeByBalanceArea/{balanceAreaId}` | GET |
| 3 | 分页查询节点列表 | `/locationMap/hierarchy/pageList` | POST |
| 4 | 根据父节点ID查询子节点列表 | `/locationMap/hierarchy/listByParentId/{parentId}` | GET |
| 5 | 根据节点类型查询节点列表 | `/locationMap/hierarchy/listByNodeType/{nodeType}` | GET |
| 6 | 查询节点详情 | `/locationMap/hierarchy/detail/{id}` | GET |
| 7 | 新增节点（支持库房+货架批量创建） | `/locationMap/hierarchy/add` | POST |
| 8 | 修改节点 | `/locationMap/hierarchy/update` | POST |
| 9 | 删除节点（级联删除子节点） | `/locationMap/hierarchy/delete/{id}` | POST |
| 10 | 批量创建货架 | `/locationMap/hierarchy/batchCreateShelf` | POST |

## 附录：详细参数说明

### 节点类型 (nodeType)
| 值 | 说明 |
|:---:|------|
| 1 | 平衡区 |
| 2 | 库房 |
| 3 | 货架 |
| 4 | 行 |
| 5 | 列 |

### 位置状态 (status)
| 值 | 说明 |
|:---:|------|
| 0 | 空闲 |
| 1 | 占用 |
| 2 | 锁定 |

### 库房类型 (warehouseType)
| 值 | 说明 |
|:---:|------|
| 0 | 新库 |
| 1 | 老库 |

### 平衡区类型 (type)
| 值 | 说明 |
|:---:|------|
| 本地 | 本地 |
| 代存 | 代存 |

## 通用请求/响应结构

```json
{
  "currentPage": 1,
  "pageSize": 10
}
```

```json
{
  "list": [],
  "pagination": {

    "pageSize": 10,
```
