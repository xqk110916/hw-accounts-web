# R01 模板打印 — 实施方案

> **版本**：v1.0  
> **创建日期**：2026-06-19  
> **所属模块**：表单管理 → 表单报送 (`src/views/formManagement/reportSubmit`)

---

## 1. 需求背景

在「表单报送」R01 界面中，现有「导出 Excel」按钮走的是后端 API 导出（`/busin/form/report/r01/export/{id}`）。

新需求要求 **新增一个「模板打印」按钮**，实现：

1. 读取预置的 Excel 模板文件 `R01-表单.xlsx`
2. 将界面中 R01 表单区域已填写的表头字段数据，填充到模板对应的单元格中
3. 生成并下载填充好数据的 `.xlsx` 文件

> **注意**：不修改现有「导出 Excel」按钮逻辑，其他报表（R03/R04 等）也不受影响。

---

## 2. 技术方案总览

| 项目 | 方案 |
| --- | --- |
| **实现方式** | 纯前端方案，不依赖后端接口 |
| **Excel 操作库** | `ExcelJS`（需新安装） |
| **文件下载** | 复用项目已有的 `file-saver@2.0.5` |

### 4.3 核心代码设计

#### 4.3.1 `exportR01Template.js` — 模板填充 & 导出模块

```javascript
import ExcelJS from 'exceljs'
import { saveAs } from 'file-saver'

/**
 * R01 表单模板字段 → Excel 单元格映射配置
 * key: templateData 中的字段名
 * value: Excel 单元格坐标（合并区域写首格坐标即可）
 */
const CELL_MAP = {
  // 发方信息
  senderUnitName:   'F3',
  senderUnitCode:   'F4',
  senderLicenseNo:  'H4',
  sendDate:         'D5',
  sendLocation:     'D6',
  // 收方信息
  receiverUnitName:  'M3',
  receiverUnitCode:  'M4',
  receiverLicenseNo: 'O4',
  receiveDate:       'K5',
  receiveLocation:   'K6',
  // 报表信息
  reportNo:          'F11',
  postUnit:          'M11',
  // 制表人
  senderTabulator:   'E15',
  receiverMaker:     'O15',
}

/**
 * 可选覆盖字段（仅当界面有值时才覆盖模板默认值）
 */
const OPTIONAL_CELL_MAP = {
  formNo:  'D2',
  format:  'N2',
}

/**
 * 将界面 R01 表单数据填充到 Excel 模板并触发下载
 * @param {Object} templateData - R01 表单界面的 templateData 对象
 */
export async function exportR01WithTemplate(templateData = {}) {
  // 1. fetch 模板文件
  const templateUrl = `${process.env.BASE_URL || '/'}templates/R01-表单.xlsx`
  const response = await fetch(templateUrl)
  if (!response.ok) throw new Error(`模板文件加载失败: ${response.status}`)
  const arrayBuffer = await response.arrayBuffer()

  // 2. ExcelJS 加载
  const workbook = new ExcelJS.Workbook()
  await workbook.xlsx.load(arrayBuffer)
  const worksheet = workbook.getWorksheet('Sheet1') || workbook.worksheets[0]

  // 3. 填充必填字段
  for (const [field, cell] of Object.entries(CELL_MAP)) {
    const value = templateData[field]
    // senderTabulator 可能存储在 senderMaker 字段中
    const actualValue = field === 'senderTabulator'
      ? (value || templateData.senderMaker || '')
      : (value || '')
    if (actualValue) {
      worksheet.getCell(cell).value = actualValue
    }
  }

  // 4. 填充可选覆盖字段
  for (const [field, cell] of Object.entries(OPTIONAL_CELL_MAP)) {
    const value = templateData[field]
    if (value) {
      worksheet.getCell(cell).value = value
    }
  }

  // 5. 生成 Blob & 下载
  const buffer = await workbook.xlsx.writeBuffer()
  const blob = new Blob([buffer], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
  })
  const fileName = `R01材料交接统计报表_${templateData.reportNo || new Date().toISOString().slice(0, 10)}.xlsx`
  saveAs(blob, fileName)
}
```

// 新增 import
import { exportR01WithTemplate } from './components/exportR01Template.js'

// methods 中新增
async handleTemplatePrint() {
  if (!this.templateData || !Object.keys(this.templateData).length) {
    this.$message.warning('请先加载或填写报表数据')

  }

},

## 6. 注意事项

### 6.1 ExcelJS 与合并单元格

ExcelJS 对合并单元格的操作规则：**只需对合并区域的左上角首格赋值**，其余从属单元格会自动继承。模板中已有合并区域设定，填充时直接操作首格坐标即可，无需手动合并/拆分。

### 6.2 样式保持

ExcelJS 使用 `load` 方式加载现有 xlsx 模板时，**会保留原有的字体、边框、对齐、合并、行高列宽等格式**。填充数据时只修改 `cell.value`，不要改动 `cell.style`，即可保证导出文件与原始模板格式完全一致。

### 6.3 `process.env.BASE_URL`

Vue CLI 项目中 `process.env.BASE_URL` 对应 `vue.config.js` 中的 `publicPath`，默认为 `'/'`。使用它拼接模板路径可以兼容不同部署路径。

### 6.4 兼容性

- `ExcelJS` 依赖 `Buffer` polyfill，Webpack 5 默认不提供。Vue CLI 5 内置了 `node-polyfill-webpack-plugin`，通常能自动处理。如果构建报错 `Buffer is not defined`，需要在 `vue.config.js` 中添加 polyfill 配置。
- `file-saver` 的 `saveAs` 在所有现代浏览器中均可正常工作。
