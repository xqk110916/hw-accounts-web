# 位置图管理 2D/3D 库房位置图需求设计文档

## 1. 背景与目标

### 1.1 背景

当前系统中，“库房管理”已有库房位置图的 2D/3D 展示能力，相关页面和组件主要位于：

- `src/views/warehouse/warehouse/index.vue`
- `src/views/warehouse/warehouse/components/WarehouseInterior3D.vue`
- `src/views/warehouse/warehouse/components/LocationMap2D.vue`
- `src/views/warehouse/warehouse/components/ShelfView3D.vue`

“位置图管理”目前以层级树方式维护平衡区、库房、列/货架、排、层等节点，页面位于：

- `src/views/warehouse/locationDrawing/index.vue`
- `src/views/warehouse/locationDrawing/components/LocationAddDialog.vue`
- `src/views/warehouse/locationDrawing/components/NodeDetailDrawer.vue`

现需要在“位置图管理”中补充查看库房位置图的能力，并对 2D 平面图进行布局编辑增强：支持拖动货架位置、支持新增“过道”概念；3D 仅展示，不参与编辑。同时，“库房管理”中的展示需要读取并呈现同一套布局结果。

### 1.2 建设目标

1. 在“位置图管理”中，库房节点支持打开“查看位置图”。
2. 查看位置图支持 2D / 3D 切换。
3. 2D 模式支持货架拖动摆放，并按网格对齐。
4. 2D 模式支持将网格单元设置为过道。
5. 2D 布局保存后，“库房管理”中的库房位置图同步展示最新布局。
6. 3D 模式保持现有展示逻辑，不做编辑能力，仅根据已有数据展示。
7. 不使用假数据进行功能联调，优先使用真实后端接口；如需临时模拟必须单独确认。

## 2. 术语与层级说明

### 2.1 当前层级

结合现有接口和用户提供的参考数据，当前树结构大致为：

```text
平衡区 nodeType=1
└── 库房 nodeType=2
    └── 列/货架 nodeType=3，例如【列】S1
        └── 排 nodeType=4，例如【排】01
            └── 层 nodeType=5，例如【层】01
```

说明：历史接口文档中存在“nodeType=3 为排 / 货架”等表述不完全一致的情况，但从当前参考数据和前端代码看，`nodeType=3` 实际承载的是 `【列】S1` 这一类货架级节点。本文档建议前端统一将其视为“货架/列”，展示名称可沿用后端返回的 `nodeName`。

### 2.2 新增概念

| 概念 | 说明 |
| --- | --- |
| 网格单元 | 2D 平面图的最小布局单位，用于描述库房平面占位。 |
| 货架占位 | 一个货架在 2D 网格中占用的矩形区域，可由货架尺寸决定宽高。 |
| 过道 | 不可放置货架的通行区域，可占用一个或多个网格单元。 |
| 布局方案 | 某个库房下货架位置、货架尺寸、过道网格、画布尺寸等配置的集合。 |

## 3. 功能范围

### 3.1 本期包含

1. 位置图管理新增库房节点“查看位置图”入口。
2. 新增位置图查看弹窗或页面，包含 2D / 3D 切换。
3. 2D 平面图支持：
   - 基于网格渲染库房平面；
   - 货架按真实层级数据生成；
   - 货架拖拽移动；
   - 货架拖动后自动吸附到网格；
   - 货架间不可重叠；
   - 货架不可放置到过道；
   - 支持设置和取消过道；
   - 支持保存、取消、重置布局。
4. 库房管理同步展示保存后的 2D 布局。
5. 3D 视图继续使用现有 Three.js 展示逻辑，不提供拖拽和过道编辑。

### 3.2 本期不包含

1. 3D 货架位置编辑。
2. 复杂路径规划、最短过道路径、消防通道规则自动校验。
3. 多版本布局审批。
4. 货架旋转、异形货架、多边形区域。
5. 跨库房拖动货架。

## 4. 业务流程设计

### 4.1 位置图管理查看流程

```text
进入位置图管理
→ 展示平衡区/库房/货架/排/层树
→ 点击库房节点“查看位置图”
→ 加载库房层级数据与位置图数据
→ 加载库房布局配置
→ 进入 2D/3D 位置图视图
```

### 4.2 2D 布局编辑流程

```text
打开库房位置图
→ 切换到 2D
→ 点击“编辑布局”
→ 拖动货架或设置过道
→ 前端实时校验占位冲突
→ 点击“保存”
→ 调用布局保存接口
→ 刷新 2D 展示
→ 库房管理读取同一布局并同步展示
```

### 4.3 2D 只读展示流程

```text
打开库房位置图
→ 获取布局配置
→ 获取位置占用数据
→ 按布局渲染货架和过道
→ 展示货架库存占用状态、空闲/占用/锁定状态
```

## 5. 页面与交互设计

### 5.1 位置图管理入口

在“位置图管理”树中，库房节点当前已有“详情”和“编辑”按钮，建议新增：

- 按钮名称：`位置图`
- 图标：`el-icon-s-grid` 或 `el-icon-map-location`
- 显示条件：`nodeType === 2`
- 点击行为：打开库房位置图视图

建议按钮顺序：

```text
位置图 / 详情 / 编辑 / 删除
```

### 5.2 位置图视图容器

建议使用全屏弹窗或右侧大抽屉，不建议复用当前较窄的 `NodeDetailDrawer`，因为 2D 拖拽画布需要足够操作空间。

推荐结构：

```text
顶部栏：库房名称、库房编号、视图切换、编辑/保存/取消
左侧或顶部工具栏：选择工具、设置过道、橡皮擦、缩放、重置布局
中间：2D 网格画布 / 3D 展示区域
右侧信息面板：选中货架详情、占用统计、图例
```

### 5.3 2D 模式交互

#### 5.3.1 查看态

1. 默认展示保存后的布局。
2. 货架展示名称、容量占用、状态颜色。
3. 鼠标悬停展示货架信息：
   - 货架名称；
   - 排数；
   - 层数；
   - 总库位；
   - 已占用；
   - 锁定数。
4. 点击货架后，右侧展示该货架下排/层/库位详情。
5. 过道以浅灰或斜纹背景展示，文案可显示“过道”。

#### 5.3.2 编辑态

1. 点击“编辑布局”进入编辑态。
2. 编辑态下允许拖动货架。
3. 拖动时显示占位预览：
   - 合法位置：绿色边框；
   - 非法位置：红色边框。
4. 松手后自动吸附到最近网格。
5. 若目标区域与其他货架或过道冲突，货架回到原位置并提示。
6. 可选择“过道工具”，点击或拖选网格设置为过道。
7. 可选择“橡皮擦工具”，取消过道。
8. 点击“保存”前做整体验证。

#### 5.3.3 约束规则

| 规则 | 说明 |
| --- | --- |
| 边界约束 | 货架不能超出库房网格范围。 |
| 货架冲突 | 不同货架占用网格不可重叠。 |
| 过道冲突 | 货架不能占用过道网格。 |
| 货架完整性 | 后端已有货架节点不能在布局中被删除，只能调整位置。 |
| 数据一致性 | 保存布局不修改货架、排、层的业务层级，只修改平面布局元数据。 |

### 5.4 3D 模式交互

1. 3D 仅展示现有库房和货架结构。
2. 保留现有旋转、缩放、点击货架查看能力。
3. 不展示编辑按钮或在 3D 下禁用编辑。
4. 若后续需要让 3D 展示过道，可作为二期；本期 3D 不强制展示过道。

### 5.5 库房管理同步展示

“库房管理”中已有 2D / 3D 展示，本次改造后：

1. 2D 展示读取同一份布局配置。
2. 位置图管理保存布局后，库房管理再次进入该库房应展示新布局。
3. 库房管理中的 2D 默认只读，不提供拖动编辑入口。
4. 库房管理中的 3D 仍按现有方式展示。

## 6. 数据设计

### 6.1 推荐方案：布局元数据独立存储

建议不要把“过道”伪装成一个货架、排或层节点，因为过道不是库存承载实体，也不应参与入库、出库、库位占用统计。推荐新增独立的布局配置数据，按库房维度存储。

### 6.2 布局数据结构

```json
{
  "warehouseId": "2046423531086733314",
  "version": 1,
  "grid": {
    "rows": 12,
    "cols": 18,
    "cellWidth": 1,
    "cellHeight": 1
  },
  "shelves": [
    {
      "shelfId": "2046423531439054850",
      "shelfCode": "S1",
      "x": 1,
      "y": 1,
      "w": 2,
      "h": 5,
      "direction": "horizontal"
    }
  ],
  "aisles": [
    { "x": 0, "y": 0, "w": 18, "h": 1 },
    { "x": 3, "y": 1, "w": 1, "h": 10 }
  ],
  "updateTime": "2026-04-26 00:00:00",
  "updateUname": "管理员"
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `warehouseId` | 库房节点 ID，对应 `nodeType=2`。 |
| `grid.rows` / `grid.cols` | 库房平面网格行列数。 |
| `cellWidth` / `cellHeight` | 单元格实际尺寸，默认可按 1 米处理。 |
| `shelves` | 货架布局，占位矩形不改变原层级。 |
| `shelfId` | 货架/列节点 ID，对应样例数据中的 `nodeType=3`。 |
| `x` / `y` | 货架左上角网格坐标。 |
| `w` / `h` | 货架占用网格宽高。 |
| `aisles` | 过道占位矩形。 |
| `version` | 布局版本号，用于并发保存校验。 |

### 6.3 货架尺寸推导

用户提出“根据货架大小，占位不同”的想法是合理的。建议本期采用以下规则：

1. 若后端返回货架尺寸字段，则优先使用后端尺寸。
2. 若无尺寸字段，则根据货架排数、层数推导默认占位：
   - `w` 默认取 2；
   - `h` 默认取排数；
   - 层数不影响平面占地，只影响货架容量和 3D 高度。
3. 对参考数据中的库房：
   - S1 至 S6 每个货架有 5 排、3 层；
   - 默认可生成 `w=2, h=5` 的货架；
   - 初始自动排布时可按 `S1 S2 S3` 一组、`S4 S5 S6` 一组，中间留一列过道。

### 6.4 首次无布局时的自动布局

如果库房还没有保存过布局，前端应根据层级树自动生成初始布局：

```text
1. 读取库房下 nodeType=3 的货架列表。
2. 根据每个货架下 nodeType=4 的排数推导货架高度。
3. 根据货架数量计算网格行列。
4. 每两个或三个货架之间自动预留过道。
5. 生成后展示为“未保存布局”状态，用户可直接保存。
```

初始布局只是基于真实层级数据的自动排布，不属于假数据。

## 7. 接口设计建议

### 7.1 现有可复用接口

| 用途 | 接口 |
| --- | --- |
| 获取层级树 | `GET /busin/locationMap/hierarchy/tree` |
| 获取节点详情 | `GET /busin/locationMap/hierarchy/detail/{id}` |
| 查看位置图 | `POST /busin/locationMap/positionMap` |

查看位置图请求：

```json
{
  "nodeId": "库房ID",
  "nodeType": "2"
}
```

### 7.2 新增布局接口建议

#### 7.2.1 查询库房 2D 布局

```text
GET /busin/locationMap/layout/detail/{warehouseId}
```

返回：

```json
{
  "code": 200,
  "data": {
    "warehouseId": "2046423531086733314",
    "version": 1,
    "grid": { "rows": 12, "cols": 18, "cellWidth": 1, "cellHeight": 1 },
    "shelves": [],
    "aisles": []
  }
}
```

若未配置布局，`data` 可返回 `null`，由前端自动生成初始布局。

#### 7.2.2 保存库房 2D 布局

```text
POST /busin/locationMap/layout/save
```

请求：

```json
{
  "warehouseId": "2046423531086733314",
  "version": 1,
  "grid": { "rows": 12, "cols": 18, "cellWidth": 1, "cellHeight": 1 },
  "shelves": [
    { "shelfId": "2046423531439054850", "shelfCode": "S1", "x": 1, "y": 1, "w": 2, "h": 5 }
  ],
  "aisles": [
    { "x": 3, "y": 1, "w": 1, "h": 10 }
  ]
}
```

保存校验建议由前后端同时执行：

1. 货架 ID 必须属于当前库房。
2. 所有货架不能超出网格。
3. 货架之间不能重叠。
4. 货架不能与过道重叠。
5. `version` 不一致时返回并发冲突，提示用户刷新后重试。

### 7.3 兼容方案

如果后端短期不新增接口，可临时将布局 JSON 挂在库房节点 `extra` 字段中，通过现有 `updateHierarchyNode` 保存。但此方案有风险：

1. `extra` 字段结构当前未形成稳定契约。
2. 容易和库房基础信息编辑互相覆盖。
3. 不利于并发控制和后续扩展。

因此仅建议作为过渡方案，正式实现仍建议新增独立布局接口。

## 8. 前端设计建议

### 8.1 组件拆分

建议新增或改造以下组件：

| 组件 | 位置建议 | 职责 |
| --- | --- | --- |
| `WarehousePositionMapDialog.vue` | `src/views/warehouse/locationDrawing/components/` | 位置图管理中的 2D/3D 查看容器。 |
| `WarehouseGridMap2D.vue` | `src/views/warehouse/warehouse/components/` 或公共业务组件目录 | 统一 2D 网格展示和编辑。 |
| `WarehouseLayoutToolbar.vue` | 可选 | 编辑工具栏：选择、过道、橡皮擦、缩放、保存。 |
| `layoutTransform.js` | `src/views/warehouse/warehouse/utils/` | 层级树、位置图数据、布局数据之间的转换。 |

### 8.2 与现有组件关系

1. `WarehouseInterior3D.vue`
   - 保留 3D 展示。
   - 2D 部分建议替换为统一的 `WarehouseGridMap2D`，避免位置图管理和库房管理各写一套。

2. `LocationMap2D.vue`
   - 当前按 `shelf.position.x/y` 绝对定位展示货架。
   - 可作为改造基础，增强网格背景、过道、拖拽、冲突校验。

3. `locationDrawing/index.vue`
   - 新增库房节点“位置图”按钮。
   - 打开 `WarehousePositionMapDialog`。

4. `warehouse/index.vue`
   - 加载库房数据时，同时加载布局配置。
   - 2D 展示使用保存后的布局。

### 8.3 拖拽实现建议

项目已有 `vuedraggable` 依赖，但它更适合列表拖拽，不适合自由画布拖动。建议本功能使用原生 Pointer Events 或鼠标事件实现：

1. `mousedown` 记录货架原始坐标。
2. `mousemove` 计算鼠标偏移并换算为网格坐标。
3. 拖动中持续做合法性校验。
4. `mouseup` 时吸附到网格并提交本地状态。

这样可以更准确控制网格吸附、冲突检测和画布滚动。

### 8.4 核心校验算法

统一将货架和过道都转为矩形占位：

```text
rect = { x, y, w, h }
```

两个矩形是否重叠：

```text
不重叠条件：
a.x + a.w <= b.x
或 b.x + b.w <= a.x
或 a.y + a.h <= b.y
或 b.y + b.h <= a.y

若以上均不满足，则重叠。
```

保存前校验：

```text
for each shelf:
  校验边界
  校验与其他 shelf 不重叠
  校验与 aisle 不重叠
```

## 10. 异常与提示

| 场景 | 提示 |
| --- | --- |
| 获取库房层级失败 | `获取库房位置结构失败，请稍后重试` |
| 获取布局失败 | `获取库房布局失败，请稍后重试` |
| 未配置布局 | `当前库房暂无布局，已根据货架结构生成初始布局` |
| 拖动到非法区域 | `该位置与货架或过道冲突` |
| 保存并发冲突 | `布局已被其他用户修改，请刷新后重试` |
| 保存成功 | `布局保存成功` |

## 11. 验收标准

### 11.1 位置图管理

1. 库房节点展示“位置图”入口。
2. 点击入口可以打开库房位置图。
3. 2D 能基于真实层级数据展示 S1、S2 等货架。
4. 2D 能展示过道。
5. 编辑态可拖动货架并保存。
6. 非法拖动会阻止保存或回退位置。
7. 保存后关闭再打开，布局仍保持。
8. 3D 能正常展示，且不出现编辑入口。

### 11.2 库房管理

1. 进入库房管理对应库房后，2D 展示与位置图管理保存结果一致。
2. 库房管理 2D 不提供布局编辑操作。
3. 库房管理 3D 原有交互不受影响。

### 11.3 数据一致性

1. 保存布局不会增删改货架、排、层节点。
2. 保存过道不会生成库存位置。
3. 位置占用统计仍以 `/locationMap/positionMap` 返回数据为准。

## 12. 测试建议

1. 无布局库房首次打开：应自动生成布局。
2. 有布局库房再次打开：应加载保存布局。
3. 拖动货架到空白网格：允许。
4. 拖动货架覆盖其他货架：禁止。
5. 拖动货架覆盖过道：禁止。
6. 新增过道覆盖货架：禁止或提示先移动货架。
7. 取消过道后可放置货架。
8. 切换 2D/3D 多次，不丢失未保存编辑状态，离开前应提示。
