### 2.4 选择容器对话框 — 3 个 Tab 改造

**文件**：`src/views/task/outbound/components/SelectContainerDialog.vue`

#### 2.4.1 Tab1「位置图」— 添加容器选中交互

**改动**：
- 删除 `detailVisible`、`detailContainer` 数据及中间详情弹窗（第 166-221 行模板 + `openContainerDetail`/`addDetailContainer`/`canSelectContainer` 方法）
- 修改 `@container-click` 处理：点击容器 → toggle 选中/取消，直接操作 `selectedContainers`

```javascript
// 新逻辑
handleContainerClick(container) {
  const normalized = this.normalizeGoods(container)
  if (!normalized.containerCode) return
  const idx = this.selectedContainers.findIndex(c => c.containerCode === normalized.containerCode)
  if (idx >= 0) {
    this.selectedContainers.splice(idx, 1)   // 已选中 → 移除
  } else {
    this.selectedContainers.push(normalized)  // 未选中 → 添加
  }
}
```

## M3：位置移动

### 3.1 新增「容器号」查询入口

**文件**：`src/views/task/move/components/detail.vue`

**改动**：明细编辑弹窗第一行新增容器号查询

```
┌─────────────────────────────────────────────┐
│ 容器号: [________]  [查询]                    │  ← 新增
├─────────────────────────────────────────────┤
│ 原库房: [下拉选择]    原位置: [下拉选择]       │
│ 目标库房: [下拉选择]   目标位置: [下拉选择]    │
└─────────────────────────────────────────────┘
```

**行为**：
- 输入容器号 → 点击查询 → 调用假接口 `getContainerInfo(containerCode)` → 返回容器所在库房 ID 和位置 ID → 自动回填「原库房」和「原位置」
- 查询失败 → toast "未找到该容器号对应的位置信息"
- 用户修改「原库房」或「原位置」→ **立即清空**容器号输入框

### 3.2 新增假接口

## M5：库房管理

### 5.1 3D 房子模型改为平房

**文件**：
- `src/views/warehouse/warehouse/components/BalanceArea3D.vue`（平衡区 3D 场景）
- `src/views/warehouse/warehouse/components/WarehouseScene3D.vue`（库房 3D 场景）

**改动**：

| 项目 | 改前 | 改后 |
|------|------|------|
| 屋顶 | 尖顶（金字塔/ConeGeometry） | **平顶**（去掉尖顶几何体） |
| 建筑高度 | 当前值 | **当前值的 2/3** |

| 建筑颜色 | 平衡区各不同色 / 库房 5 色循环 | **统一灰色 `#909399`** |

### 5.2 2D/3D 货架平面尺寸写死 2×5

**文件**：
- `src/views/warehouse/warehouse/components/WarehouseGridMap2D.vue`（2D 货架图）
- `src/views/warehouse/warehouse/components/WarehouseInterior3D.vue`（库房内部 3D）
- `src/views/warehouse/warehouse/components/ShelfView3D.vue`（单货架 3D）

**改动**：

- 货架类型编码 `6-5-2-10` 中 `2`= 宽度 2m、`10`= 长度 10m
- 渲染时忽略这两个参数，所有货架平面**写死 2m×5m**（格子 5 格 × 2 格）
- 不再读取类型编码的 parts[2]/parts[3] 来决定矩形尺寸

### 5.3 容器颜色逻辑 — 状态着色

**文件**：
- `src/views/warehouse/warehouse/utils/colorHelper.js`
- `src/views/warehouse/warehouse/components/WarehouseGridMap2D.vue`
- `src/views/warehouse/warehouse/components/WarehouseInterior3D.vue`
- `src/views/warehouse/warehouse/components/ShelfView3D.vue`

**改动**：

- 废弃 `getColorByDate(dateString)` 哈希算法
- 新增 `getContainerColor(status, materialCode)` 函数

| 状态 | 颜色 | 色值 | 判断条件 |
|------|------|------|----------|
| 🟡 占用锁定 | 黄色（warning） | `#E6A23C` | `status === '2'` |
| 🟢 有物料 | 绿色（success） | `#67C23A` | `materialCode` 非空 |
| 🔵 空闲 | 白色（primary） | `#fff` | 其他情况 |

> 注：`WarehouseGridMap2D.vue` 的 2D 位置图中，空闲容器色块及底部图例显示为白色 `#fff`；3D 视图（`WarehouseInterior3D.vue`、`ShelfView3D.vue`）仍按上表使用蓝色 `#409EFF`。

### 5.4 2D 库房图 — 排间空隙消除

**改动**：去除默认情况下排与排之间的空隙，默认紧挨在一起（grid gap/border-spacing 等相关样式改为 0）。

### 5.5 2D 库房图 — 过道设置改为表单

**文件**：`src/views/warehouse/warehouse/components/WarehouseGridMap2D.vue`

**改动**：

**删除**：
- 过道工具栏按钮（涂抹工具）
- `handleFloorMouseDown` 涂抹相关逻辑
- 旧 `layout.aisles[{x,y,w,h}]` 坐标数据（直接废弃，不做迁移）

**新增**：
- 「过道设置」表单面板，数据结构：

```javascript
aisleSettings: {
  rows: [
    { afterIndex: 1, width: 2 },  // 第 1 排后面有过道，宽 2 格
    { afterIndex: 3, width: 2 },  // 第 3 排后面有过道，宽 2 格
  ],
  cols: [
    { afterIndex: 1, width: 2 },  // 第 1 列后面有过道，宽 2 格
  ]
}
```

**默认规则**（自动生成）：
- **奇数排/列**（如 5）：`1 | 2,3 | 4 | 5`（两边单，中间每 2 个一组）
- **偶数排/列**（如 6）：`1 | 2,3 | 4,5 | 6`（两边单，中间每 2 个一组）

**UI**：
- 显示当前过道列表，支持删除
- 「新增排间过道」/「新增列间过道」按钮
- 2D 图中即时预览
- 过道宽度单位：格子数（与货架 2D 展示的格子图一致，货架 2m×10m 占 10 格(x)×2 格(y)）

### 5.6 导出位置图 — 添加库位统计信息

**文件**：`src/views/warehouse/warehouse/components/WarehouseGridMap2D.vue`

## 实施顺序建议

```
Phase 1: M1(入库管理) → 改动最小，快速完成
Phase 2: M3(位置移动) → 独立模块，无依赖
Phase 3: M2(出库管理) → 改动密集，需仔细测试交互
Phase 4: M4(位置图管理) → 依赖库房图组件，建议先于 M5 或与 M5 并行
Phase 5: M5(库房管理) → 改动最深，涉及 2D/3D 底层，最后执行
```
