# 任务管理模块设计文档

## 1. 页面概述

任务管理模块包含四个子页面：**入库管理**、**出库管理**、**位置移动**、**实物盘存**。所有页面均为标准 CRUD 列表页，包含筛选搜索、数据表格、分页、新增/编辑/详情弹窗、审核流程。

### 1.1 整体排序规则
> **批注：** 排序规则：按库房排序

## 3. 交互流程

### 3.1 入库管理流程

```
用户操作                    系统处理
  │
  ├─ 筛选查询 ──────────→ 后端返回分页数据
  │                         ↓
  ├─ 添加任务 ──────────→ 打开新增弹窗
  │                         │
  │                         ├─ 选择调拨依据（联动加载）
  │                         ├─ 填写基本信息
  │                         ├─ 添加明细（手动或导入）
  │                         │    ├─ 校验库位是否被占用
  │                         │    └─ 校验材料与库房产品限制
  │                         └─ 提交 → 状态变为"待确认"
  │
  ├─ 编辑（已确认数据） ─→ 提交变更审核
  │                         │
  │                         ├─ 校验调拨依据材料一致性
  │                         └─ 状态变为"变更审核中"
  │
  ├─ 审核 ──────────────→ 审核通过 → 更新库存信息
  │                         │
  │                         └─ 再次校验库位是否被占位
  │
  └─ 选择已完成的调拨依据 → 提示"是否使用？"
```

### 3.2 出库管理流程
与入库类似，增加自动配置/手工选择位置

### 3.3 位置移动流程
```
添加任务 → 选择容器 → 选择目标位置 → 提交 → 审核通过 → 移动容器位置
```

### 3.4 实物盘存流程
```
添加任务 → 选择库房 → 加载位置图 → 录入盘存结果 → 提交审核
                                                    │
                                                    ├─ 如有漏盘 → 提示未盘点容器
                                                    └─ 审核通过 → 更新库存
```

## 5. 批注需求汇总

| 编号 | 页面 | 批注内容 | 实现要求 |
|------|------|---------|---------|
| 1 | 首页 | 排序规则：按库房排序 | 列表默认排序按库房字段 |
| 2 | 入库-列表 | 添加字段如列表 | 列表字段按需添加 |
| 3 | 入库-编辑 | 已确认的在修改需要提交审核，审核通过后数据才更新 | 已确认状态的数据修改走变更审核流程 |
| 4 | 入库-编辑 | 校验调拨依据的材料和入库材料是否一致，不一致的提示 | 提交时校验材料一致性 |
| 5 | 入库-编辑 | 校验入库的材料和库房的产品限制是否匹配，不匹配的提示 | 提交时校验库房限制 |
| 6 | 入库-提交 | 再次校验库位是否被占位，审核通过后更新库存信息 | 审核通过前二次校验库位 |
| 7 | 入库-提交 | 提交时判断库位状态是否被占用 | 提交时校验库位 |
| 8 | 入库-选择 | 选择已完成的，给出提示，是否使用？ | 选择已完成调拨依据时二次确认 |
| 9 | 入库-编号 | 格式为年+编号（001开始递增），入口出库共用 | 任务编号自动生成规则 |
| 10 | 入库-调拨 | 单选 | 调拨依据选择为单选 |
| 11 | 入库-联动 | 联动调拨依据 | 调拨依据与材料联动 |
| 12 | 出库-容器 | 容器号、封记号、生产单位、入库时间、重量（毛、皮、净） | 出库列表展示字段 |
| 13 | 出库-多选 | 多选 | 出库容器号筛选支持多选 |
| 14 | 出库-容器2 | 容器号、封记号、生产单位、入库时间 | 出库容器筛选字段 |
| 15 | 位置移动 | 通过后，移动容器位置 | 审核通过后执行位置移动 |
| 16 | 实物盘存-表 | 一个库房对应一个表+位置图 | 盘存任务与库房、位置图关联 |
| 17 | 实物盘存-提交 | 结果录入完成后可提交审核 | 盘存完成后可提交 |
| 18 | 实物盘存-漏盘 | 盘存完成如有漏盘，系统提示有未盘点容器 | 提交时校验漏盘 |
| 19 | 实物盘存-不正常 | 不正常问题必输 | 状态为不正常时备注必填 |
| 20 | 实物盘存-排序 | 按位置排序 | 明细列表按位置排序 |

## 7. 接口/数据结构说明

### 7.1 任务编号生成接口
```
POST /task/generateNo
参数: { type: 'inbound'|'outbound'|'move'|'inventory' }
返回: { code: 1, data: { taskNo: '2025001' } }
```

### 7.2 列表查询接口
```
POST /task/{module}/listPage
参数: { currentPage, pageSize, taskNo, status, ...筛选条件 }
返回: { code: 1, data: { list: [], pagination: { total, currentPage, pageSize } } }
```

### 7.3 详情查询接口
```
POST /task/{module}/getInfoById
参数: { id }
返回: { code: 1, data: { ...基本信息, details: [...明细列表], changeLogs: [...] } }
```

### 7.4 新增/编辑接口
```
POST /task/{module}/saveOrUpdate
参数: { id?, taskNo, allocationBasis, ...基本信息, details: [...] }
返回: { code: 1, data: null }
```

### 7.5 删除接口
```
POST /task/{module}/delete
参数: { ids }
返回: { code: 1, data: null }
```

### 7.6 审核接口
```
POST /task/{module}/audit
参数: { id, result: 'pass'|'reject', remark? }
返回: { code: 1, data: null }
```

### 7.7 导入接口
```
POST /task/{module}/import
参数: FormData { file, type: '1'|'2' }
返回: { code: 1, data: { successCount, failCount, failInfo: [...] } }
```

### 7.8 变更审核接口
```
POST /task/{module}/submitChangeAudit
参数: { id, changes: [...] }
返回: { code: 1, data: null }
```

### 7.9 调拨依据列表接口
```
POST /allocation/listPage
参数: { currentPage, pageSize, ... }
返回: { code: 1, data: { list: [{ value, label, materialList: [] }], pagination } }
```

### 7.10 库房列表接口
```
POST /warehouse/list
参数: {}
返回: { code: 1, data: { list: [{ value, label }] } }
```
