# 系统管理接口文档

---

## 一、菜单管理

> 新增业务界面时，若涉及新目录或新功能页，需同步调用相关接口完成菜单注册，以确保导航系统正常显示。
> 示例流程：若新增模块 `task`，需先调用新增目录接口（类型 M），获取 ID 后，再在其下调用新增菜单接口（类型 C）。

### 1. 查询菜单树

- **请求方式**: `GET`
- **请求地址**: `/base/menu/treeListByClientType`

**请求参数**

| 参数名 | 类型 | 必选 | 描述 | 示例值 |
| :--- | :--- | :--- | :--- | :--- |
| `clientType` | String | 是 | 客户端类型 | `0` |

**返回示例**
```json
{
  "code": 1,
  "data": [
    {
      "id": "1782971139194662913",
      "pid": "0",
      "sortNum": 1,
      "label": "应用管理",
      "fullName": "应用管理",
      "parentId": "0",
      "enCode": "application",
      "path": "application",
      "frameFlag": null,
      "menuType": "M",
      "visibleFlag": "1",
      "perms": "",
      "icon": "component",
      "clientType": "0",
      "remark": null,
      "sysFlag": 1,
      "linkType": 0,
      "linkUrl": null,
      "children": [ {}, {}, {} ]
    }
  ]
}
```

---

### 2. 新增菜单 (目录/界面)

- **请求方式**: `POST`
- **请求地址**: `/base/menu`

**请求体参数 (JSON)**

| 字段名 | 类型 | 描述 | 注意事项 |
| :--- | :--- | :--- | :--- |
| `parentId` | String | 父级菜单 ID | 顶级菜单请填 `0` |
| `menuType` | String | 菜单类型 | `M`: 目录, `C`: 菜单 |
| `fullName` | String | 菜单显示文本 | - |
| `linkType` | Number | 链接类型 | `0`: 内部页面, `1`: 外部页面 (默认 0) |
| `path` | String | 界面对应目录 | 对应代码中的文件夹名称 |
| `linkUrl` | String | 外部链接 URL | 仅在 `linkType` 为 1 时有效 |
| `perms` | String | 权限唯一标识 | - |
| `sortNum` | Number | 排序权重 | 数值越小越靠前 |
| `visibleFlag` | String | 显示状态 | `0`: 隐藏, `1`: 显示 |
| `clientType` | String | 客户端类型 | 固定传 `0` |
| `icon` | String | 图标名称 | - |
| `enCode` | String | 英文编码 | - |

**请求体示例**
```json
{
  "parentId": "2042412186163056641",
  "menuType": "C",
  "fullName": "测试界面",
  "linkType": 0,
  "path": "test",
  "linkUrl": "",
  "perms": "test",
  "sortNum": 99,
  "visibleFlag": "1",
  "clientType": "0",
  "icon": "",
  "enCode": "csjm"
}
```

### 2. 新增字典项

- **请求方式**: `POST`
- **请求地址**: `/base/dictionary`

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| parentId | String | 是 | 父级ID，顶层传 "0"，子级传对应父级ID |
| fullName | String | 是 | 字典名称 |
| dictValue | String | 是 | 字典值 |
| bizCode | String | 是 | 业务编码 |
| sortNum | Number | 否 | 排序号，默认 99 |
| enableFlag | Number | 否 | 启用状态：1-启用 |
| description | String | 否 | 描述 |
| dictType | String | 是 | 字典类型，固定值 "category" |

**请求示例**

```json
{
  "parentId": "0",
  "fullName": "字典名称",
  "dictValue": "zdmc",
  "bizCode": "zdmc",
  "sortNum": 99,
  "enableFlag": 1,
  "description": "",
  "dictType": "category"
}
```

**响应示例**

```json
{
  "code": 1,
  "data": null,
  "msg": "操作成功",
  "success": true
}
```

### 3. 修改字典项

- **请求方式**: `PUT`
- **请求地址**: `/base/dictionary/{id}`

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 字典项ID |
| parentId | String | 是 | 父级ID |
| parentName | String | 否 | 父级名称 |
| fullName | String | 是 | 字典名称 |
| dictValue | String | 是 | 字典值 |
| bizCode | String | 是 | 业务编码 |
| sortNum | Number | 否 | 排序号 |
| enableFlag | Number | 否 | 启用状态：1-启用 |
| description | String | 否 | 描述 |
| dictType | String | 是 | 字典类型，固定值 "category" |
| color | String | 否 | 颜色 |
| defaultFlag | Number | 否 | 是否默认 |

**请求示例**

```json
{
  "id": "2046423912768405505",
  "parentId": "0",
  "parentName": "",
  "fullName": "测试",
  "dictValue": "ceshi",
  "bizCode": "ceshi",
  "sortNum": 99,
  "enableFlag": 1,
  "description": "",
  "dictType": "category",
  "color": null,
  "defaultFlag": null
}
```

**响应示例**

```json
{
  "code": 1,
  "data": null,
  "msg": "操作成功",
  "success": true
}
```
