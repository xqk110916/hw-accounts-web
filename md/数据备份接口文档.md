# 数据备份接口文档

---

## 一、数据备份管理

### 1. 查询所有备份配置

- **接口地址**: `GET /api/busin/backup/config/list`

**响应示例**
```json
{
  "code": 1,
  "data": [
    {
      "backName": "实时库存",
      "backType": "1",
      "createTime": "2026-05-29 17:07",
      "deleteFlag": 0,
      "id": "1",
      "isBack": "1",
      "newTime": "2026-05-27 00:00"
    }
  ],
  "msg": "操作成功",
  "success": true
}
```

**返回字段说明 (DataBackEntity)**

| 参数名称 | 类型 | 描述 |
|---------|------|------|
| id | string | 主键ID |
| backName | string | 备份名称（实时库存、位置表、入库数据、出库数据、移库数据、盘存数据） |
| backType | string | 备份类型（1：实时库存数据 2：库位表 3：入库数据 4：出库数据 5：移库数据 6：盘存数据） |
| isBack | string | 是否备份（1：是 2：否） |
| createTime | string | 执行时间 |
| newTime | string | 数据最新日期 |
| deleteFlag | integer | 删除标识 |

---

### 2. 修改备份配置状态（is_back改为1时自动执行备份）

- **接口地址**: `PUT /api/busin/backup/config/{id}`

**路径参数**

| 参数名称 | 类型 | 必填 | 描述 |
|---------|------|------|------|
| id | long | 是 | 备份配置ID |

**请求参数**

| 参数名称 | 类型 | 必填 | 描述 |
|---------|------|------|------|
| isBack | string | 是 | 是否备份（1：是 2：否） |

**请求示例**
```
PUT /api/busin/backup/config/1?isBack=1
```

**响应示例**
```json
{
  "code": 1,
  "data": true,
  "msg": "操作成功",
  "success": true
}
```

---

### 3. 从备份恢复数据（恢复前会删除原表数据）

- **接口地址**: `POST /api/busin/backup/restore/{backType}`

**路径参数**

| 参数名称 | 类型 | 必填 | 描述 |
|---------|------|------|------|
| backType | string | 是 | 备份类型（1：实时库存数据 2：库位表 3：入库数据 4：出库数据 5：移库数据 6：盘存数据） |

**请求示例**
```
POST /api/busin/backup/restore/1
```

**响应示例**
```json
{
  "code": 1,
  "data": true,
  "msg": "操作成功",
  "success": true
}
```

---

### 4. 全量导出数据库为 DMP 文件并下载（流式传输）

- **接口地址**: `GET /api/busin/backup/export/dmp/stream`
- **接口说明**: 全量导出数据库为 DMP 文件（数据库逻辑备份格式），以流式（Stream）方式返回，前端直接触发文件下载。
- **Tag**: `data-backup-controller`

**请求参数**: 无

**响应**: 二进制文件流（`application/octet-stream`），响应头携带 `Content-Disposition: attachment; filename="xxx.dmp"`。

**前端调用示例**
```js
request({
  url: '/busin/backup/export/dmp/stream',
  method: 'get',
  responseType: 'blob'
})
```

---

### 5. 上传 DMP 文件并恢复数据库

- **接口地址**: `POST /api/busin/backup/restore/upload`
- **Content-Type**: `multipart/form-data`
- **接口说明**: 上传 DMP 备份文件，后端解析并恢复整库数据。⚠️ 恢复前会清空原表数据，属于高危操作，前端需二次确认。
- **Tag**: `data-backup-controller`

**请求参数**

| 参数名称 | 类型 | 必填 | 描述 |
|---------|------|------|------|
| file | file | 是 | DMP 备份文件 |

**请求示例**
```
POST /api/busin/backup/restore/upload
Content-Type: multipart/form-data

file: <DMP 文件>
```

**响应示例**
```json
{
  "code": 1,
  "data": true,
  "msg": "操作成功",
  "success": true
}
```

**前端调用示例**
```js
const formData = new FormData()
formData.append('file', file)
request({
  url: '/busin/backup/restore/upload',
  method: 'post',
  data: formData,
  headers: { 'Content-Type': 'multipart/form-data' }
})
```
