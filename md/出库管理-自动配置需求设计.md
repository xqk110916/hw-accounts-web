# 出库管理-出库任务自动配置需求设计

## 1. 文档目的

本文档用于说明“出库管理-添加/编辑出库任务”中“自动配置”功能的操作界面需求、接口对接方式和数据流转规则。设计依据为自动配置界面原型图以及 `md/出库管理-取货方案接口文档.md` 中的取货方案接口。

本需求只约束操作界面的字段、结构、交互和接口映射，不约束视觉样式。

## 2. 功能范围

自动配置用于在出库任务中，根据待出库物料的重量需求、库房、生产单位、入库任务编号、入库时间等条件，调用后端自动配重接口生成一个或多个取货方案。用户查看方案后，可选择某一个方案回填到出库明细表。

入口位置：

- 页面：出库管理
- 弹窗：添加出库任务 / 编辑出库任务
- 区域：明细信息
- 操作按钮：自动配置

接口范围：

- 自动生成方案：`POST /busin/pick/auto-weight`
- 出库提交：继续沿用 `POST /busin/outbound/submit`
- 出库编辑：继续沿用 `POST /busin/outbound/updateData`

## 3. 业务前置条件

点击“自动配置”前，页面至少应具备以下数据：

| 前置数据 | 来源 | 要求 |
| --- | --- | --- |
| 任务编号 | 出库任务表单 | 新增时可由批次号接口生成，自动配置不强依赖 |
| 调拨依据 | 出库任务表单 | 必须已选择，用于确定出库材料范围 |
| 材料编码/物料 | 调拨依据或表单材料范围 | 至少存在 1 条待出库物料 |
| 出库重量 | 出库任务材料需求 | 每个待出库物料应明确毛重、皮重、净重中的至少一个目标值 |
| 收方单位、密级、出库人 | 出库任务表单 | 不作为自动配置必填条件，但提交出库任务时仍按原表单规则校验 |

若缺少调拨依据或材料需求，点击“自动配置”时应提示用户先选择调拨依据/材料，不发起接口请求。

## 4. 自动配置弹窗结构

弹窗建议命名为“自动配置”或“自动取货方案”。整体结构分为六个区域：

1. 待出库物料条件区
2. 优先级字段区
3. 方案标签区
4. 当前方案物料满足情况区
5. 当前方案容器明细表
6. 底部操作区

### 4.1 待出库物料条件区

顶部按待出库物料分组展示，每个物料一行或一块信息。原型中展示 2 个物料，字段如下：

| 字段 | 示例 | 接口字段/来源 | 说明 |
| --- | --- | --- | --- |
| 序号 | 1、2 | 前端序号 | 自动生成 |
| 物料名称 | 物料1 | `materialName` | 取调拨依据材料名称或材料主数据名称 |
| 毛重 | 10.00000 | `grossWeight` | 目标毛重 |
| 皮重 | 0.00000 | `tareWeight` | 目标皮重 |
| 净重 | 10.00000 | `netWeight` | 目标净重 |
| 单位 | kg | `unit` | 默认取材料单位，缺省时显示 `kg` |
| 库房 | rss1110;RSS1111 | `warehouses` | 多个库房以分隔符展示 |
| 入库时间 | 2025.10.10-2026.10.10 | `startStorageTime/endStorageTime` | 用作库存入库时间筛选 |
| 生产单位 | xxxxxx;xxxxxx | `suppliers` | 多个生产单位以分隔符展示；界面统一展示为“生产单位”，接口字段沿用 `suppliers` |
| 入库任务编号 | RK2026001;RK2026002 | `batchNos` | `batchNos` 传入入库任务编号数组；响应中的 `batchNo` 也按入库任务编号展示 |

顶部右侧提供“编辑”按钮：

- 点击后进入物料条件编辑态或打开条件编辑弹窗。
- 可编辑字段为：库房、生产单位、入库任务编号、入库时间范围、目标重量、单位。
- 物料编码/物料名称原则上来自调拨依据，不建议在自动配置弹窗内新增物料。
- 保存条件后重新调用 `POST /busin/pick/auto-weight` 获取方案。

材料需求来源规则：

- 优先从当前出库任务已选调拨依据的 `goodsList` 读取材料需求，使用 `goodCode` 作为材料编码，使用 `goodWeight` 作为目标净重。
- 与当前出库表单逻辑保持一致：调拨依据通过 `_transferSelected` 维护前端选择态，提交和自动配置入参均使用 `transferId` 与 `goodCodes` 表达同一文号下的材料范围。
- 若调拨依据材料缺少 `goodWeight`，允许用户在自动配置条件编辑中补充目标净重后再发起自动配置。

### 4.2 优先级字段区

自动配置界面需要展示“优先级”字段区，位置在待出库物料条件区之后、方案标签区之前。该区域按复选框展示，用户可勾选一个或多个优先规则。

| 界面字段 | 接口字段 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 尽量一个库房 | `priorityWarehouse` | `true` | 勾选后优先生成库房数量更少的方案 |
| 尽量一个批次 | `priorityBatch` | `false` | 勾选后优先生成入库任务编号数量更少的方案 |
| 尽量一个生产单位 | `priorityManufacturer` | `false` | 勾选后优先生成生产单位数量更少的方案 |
| 入库时间先进先出 | `priorityFifo` | `false` | 勾选后优先选择入库时间更早的容器 |
| 最大方案数 | `maxSolutions` | `3` | 默认返回 3 个方案；可扩展为数字输入 |

默认值按照原型表现执行：默认勾选“尽量一个库房”，其他优先级默认不勾选。接口字段保持不变，仅按复选框勾选状态传布尔值。

### 4.3 方案标签区

接口可能返回多个方案，原型中展示为：

- 方案1
- 方案2
- 方案3
- 另外3个
- 刷新/重新计算图标

交互要求：

| 元素 | 说明 |
| --- | --- |
| 方案标签 | 每个方案对应一个 `PickPlanVO`，按 `priority` 从小到大排序展示 |
| 默认选中 | 默认选中排序第一的方案 |
| 另外 N 个 | 当方案数量超过直接展示上限时，以下拉或弹出菜单展示剩余方案 |
| 刷新/重新计算 | 使用当前条件重新调用 `POST /busin/pick/auto-weight` |
| 空方案 | 若接口返回空列表，展示“暂无可用方案”，禁用“使用本方案” |

每个方案内部应保留以下统计信息，可用于方案摘要、悬浮提示或后续扩展展示：

| 接口字段 | 说明 |
| --- | --- |
| `solutionId` | 方案 ID |
| `priority` | 方案优先级 |
| `containerCount` | 容器数量 |
| `batchCount` | 涉及入库任务编号数量 |
| `manufacturerCount` | 涉及生产单位数量 |
| `warehouseCount` | 涉及库房数量 |
| `totalGrossWeight` | 方案总毛重 |
| `totalTareWeight` | 方案总皮重 |
| `totalNetWeight` | 方案总净重 |
| `oldestStorageTime` | 最早入库时间 |

### 4.4 当前方案物料满足情况区

方案标签下方按物料展示当前方案是否满足出库重量。原型示例：

- 物料1 出库重量（毛皮净）：10.0000/0.00000/10.00000 已满足
- 物料2 出库重量（毛皮净）：10.0000/0.00000/10.00000 未满足

字段要求：

| 字段 | 来源/计算方式 | 说明 |
| --- | --- | --- |
| 物料名称 | 请求物料 `materialName` 或容器 `material` | 按物料分组 |
| 目标出库重量 | 请求物料的 `grossWeight/tareWeight/netWeight` | 格式为“毛/皮/净” |
| 方案匹配重量 | 当前方案容器按物料汇总的 `grossWeight/tareWeight/netWeight` | 用于判断满足状态 |
| 满足状态 | 前端计算 | 当方案匹配重量达到目标重量时为“已满足”，否则为“未满足” |

满足判断建议：

- 优先按净重判断：当前方案该物料净重合计 `>=` 目标净重则满足。
- 若目标净重为空但毛重有值，则按毛重判断。
- 若目标毛重、皮重、净重均为空，则该物料不参与满足判断，但应提示需求不完整。
- 重量比较需要使用数字类型，展示保留 5 位小数。

### 4.5 当前方案容器明细表

当前选中方案下方展示容器列表，表格字段参照原型和出库明细表：

| 列名 | 接口字段 | 回填出库明细字段 | 说明 |
| --- | --- | --- | --- |
| 序号 | `serialNumber` | 前端序号 | 无值时按表格顺序生成 |
| 材料编码 | `goodCode` | `goodCode` | 必填 |
| 容器号 | `containerCode` | `containerCode` | 必填 |
| 生产单位 | `productionUnit` | `productionUnit` | 无值显示空 |
| 库房 | `warehouse` | `warehouseName` | 自动配置响应字段为 `warehouse` |
| 位置 | `shelfCode/rowCode/columnCode` | `shelfCode/rowCode/columnCode` | 展示格式为“排-行-列” |
| 货箱号 | `boxNo` | `boxNum` | 字段名需转换 |
| 封记编码1 | `sealCode1` | `sealCode1` | - |
| 封记编码2 | `sealCode2` | `sealCode2` | - |
| 净重 | `netWeight` | `netWeight` | 直接展示容器净重 |
| 重量（毛、皮、净） | `grossWeight/tareWeight/netWeight` | `grossWeight/tareWeight/netWeight` | 展示格式：毛/皮/净 |
| 金属量% | `metalPercentage` | `metalPercentage` | - |
| 操作 | 前端操作 | - | 删除 |

表格操作要求：

- “删除”：仅从当前方案临时移除该容器，并重新计算物料满足情况。
- 删除后如果某个物料不再满足，应实时显示“未满足”。
- 已删除容器不影响接口原始返回；切换方案后按接口返回的方案重置显示。
- 如需恢复删除，可通过重新计算或切换方案实现。

## 5. 底部操作

底部按钮参照原型：

| 按钮 | 行为 |
| --- | --- |
| 使用本方案 | 将当前方案容器列表转换为出库明细 `detailList`，关闭自动配置弹窗 |
| 取消 | 关闭弹窗，不修改出库明细 |

“使用本方案”校验：

1. 当前方案必须存在容器列表。
2. 若存在“未满足”物料，点击时弹出二次确认；用户确认后允许强制使用当前方案并回填出库明细。
3. 容器号不可为空。
4. 同一个出库任务明细中不可出现重复容器号。
5. 若当前出库明细已存在数据，使用本方案时应提示“将覆盖当前明细，是否继续？”。

## 6. 自动配置接口对接

### 6.1 请求接口

```http
POST /busin/pick/auto-weight
```

请求体：

```json
{
  "materials": [
    {
      "goodCode": "1001",
      "materialName": "物料1",
      "netWeight": 10,
      "grossWeight": 10,
      "tareWeight": 0,
      "unit": "kg",
      "suppliers": ["生产单位1", "生产单位2"],
      "warehouses": ["RSS1110", "RSS1111"],
      "batchNos": ["RK2026001", "RK2026002"],
      "startStorageTime": "2025-10-10 00:00:00",
      "endStorageTime": "2026-10-10 23:59:59"
    }
  ],
  "priorityWarehouse": true,
  "priorityManufacturer": false,
  "priorityBatch": false,
  "priorityFifo": false,
  "maxSolutions": 3
}
```

字段映射：

| 请求字段 | 取值来源 |
| --- | --- |
| `materials[].goodCode` | 调拨依据材料编码 / 出库任务材料编码 |
| `materials[].materialName` | 调拨依据材料名称 / 材料主数据名称 |
| `materials[].grossWeight` | 当前物料目标毛重 |
| `materials[].tareWeight` | 当前物料目标皮重 |
| `materials[].netWeight` | 当前物料目标净重 |
| `materials[].unit` | 当前物料单位，默认 `kg` |
| `materials[].suppliers` | 生产单位条件；界面字段名为“生产单位”，接口字段沿用 `suppliers` |
| `materials[].warehouses` | 原型中的库房条件 |
| `materials[].batchNos` | 入库任务编号数组；取库存容器中的 `taskNum` 或响应中的 `batchNo` |
| `materials[].startStorageTime` | 入库时间范围开始 |
| `materials[].endStorageTime` | 入库时间范围结束 |
| `materials[].startDeliveryTime` | 当前版本不展示、不组装、不提交 |
| `materials[].endDeliveryTime` | 当前版本不展示、不组装、不提交 |
| `priorityWarehouse` | 尽量一个库房 |
| `priorityManufacturer` | 尽量一个生产单位 |
| `priorityBatch` | 尽量一个批次/入库任务编号 |
| `priorityFifo` | 入库时间先进先出 |
| `maxSolutions` | 最大返回方案数 |

### 6.2 响应处理

接口返回 `PickPlanVO` 列表，前端按如下结构使用：

| 响应字段 | 用途 |

| `goodCode` | `goodCode` |
| `material` | 可作为材料名称展示字段 |

## 7. 与出库提交/编辑的衔接

用户点击“使用本方案”后，自动配置结果进入出库任务明细表，后续提交仍使用出库管理原接口。

新增出库任务时：

```json
{
  "taskNum": "CK2026001",
  "transferId": 1,
  "securityLevel": "1",
  "outboundMan": "张三",
  "receiveUnit": "收方单位",
  "goodsList": [
    {
      "containerCode": "xxxx",
      "goodCode": "1001",
      "warehouseName": "RSS1-01",
      "boxNum": "20kg",
      "grossWeight": "10.00000",
      "tareWeight": "0.00000",
      "netWeight": "10.00000",
      "metalPercentage": "10",
      "productionUnit": "xxxxx",
      "shelfCode": "RSS1-01",
      "rowCode": "1",
      "columnCode": "1",
      "sealCode1": "xxxxx",
      "sealCode2": "xxxxx"
    }
  ],
  "submitType": 0
}
```

编辑出库任务时：

- 新加入的自动配置容器进入 `dtoList`。
- 已存在并被保留或调整的容器进入 `editList`。
- 被自动配置覆盖删除的原明细应写入 `goodIds` 和 `containerCodes`。
- 已确认任务发起修改时继续带 `updateType=1`，进入变更审核。

## 8. 状态与异常处理

| 场景 | 处理要求 |
| --- | --- |
| 接口请求中 | 禁用“使用本方案”和“刷新/重新计算”，展示加载状态 |
| 接口返回空方案 | 展示空状态，禁用“使用本方案” |
| 接口异常 | 提示接口错误，不清空用户已填写的条件 |
| 部分物料未满足 | 对应物料展示“未满足”；点击“使用本方案”时二次确认，确认后允许强制使用 |
| 容器重复 | 使用方案前拦截，并提示重复容器号 |
| 重量为非数字 | 自动配置前拦截，并提示修正重量 |
| 删除容器后方案不满足 | 实时更新满足状态；使用时按未满足方案二次确认 |
| 切换方案 | 使用所选方案原始容器列表重新渲染表格和满足状态 |

## 9. 权限与可用性

自动配置按钮展示规则建议与出库明细编辑能力保持一致：

| 出库任务模式 | 是否展示自动配置 |
| --- | --- |
| 新增 | 展示 |
| 暂存编辑 | 展示 |
| 待确认编辑 | 展示 |
| 已确认修改 | 可展示，使用后提交变更审核 |
| 详情查看 | 不展示 |
| 审核查看 | 不展示 |

若后端或业务要求自动配置只允许新增时使用，则编辑/修改模式可隐藏按钮；但需与手工选择保持一致的业务口径。

## 10. 已明确规则

以下规则已按当前需求固定，开发实现时不再作为开放问题处理：

| 规则 | 说明 | 实施要求 |
| --- | --- | --- |
| 生产单位字段 | 界面统一显示“生产单位”，不再使用其他业务叫法 | 自动配置请求仍使用 `materials[].suppliers` 承载生产单位数组；响应和回填优先使用 `productionUnit` |
| 净重字段 | 原型中的重量列统一按净重处理 | 容器明细表直接展示 `netWeight`，不新增 `materialWeight` 字段 |
| 入库任务编号 | `batchNo` 按入库任务编号理解 | 请求 `materials[].batchNos` 传入入库任务编号数组；响应 `batchNo` 按入库任务编号展示 |
| 未满足强制使用 | 未满足方案允许使用 | 点击“使用本方案”时二次确认，确认后回填当前方案容器 |
| 出库时间 | 当前版本不作为自动配置条件 | 不展示、不组装、不提交 `startDeliveryTime/endDeliveryTime` |
