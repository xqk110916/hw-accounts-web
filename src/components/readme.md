#### 全局组件注册规范说明

##### 1.在考虑是否需要注册全局组件时，需要从以下两个方面来衡量：

1. 当同一逻辑处理、界面交互、方法函数在 2 个或以上页面中出现时，需要将其单独拆分出来，当为全局组件来处理并维护；
2. 当一段逻辑比较复杂，圈复杂度较高时，需要拆分其功能逻辑，保证单一职责，高内聚低耦合，当为全局逻辑来处理； PS：一般此文件夹处的组件为全局 UI 组件或全局业务组件；

##### 2. 全局组件的命名

一个组件应当维护在一个文件夹内，命名尽可能由简短的英文单词组成（大驼峰方式），组件文件夹内一般以 index.vue 为组件入口文件，其依附的组件可按实际所需同级维护/放置于 components 文件夹下；

##### 3. 全局组件的维护

组件应保证其单一职责、外部无依赖、尽可能使用动态配置、动态参数等保证其灵活性与兼容性。
